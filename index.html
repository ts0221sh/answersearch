使い方

単語ペア+コメントの蓄積データから、特定の文字対応を持つペアを絞り込み検索するツール。

データソース

  • GAS Web App URL: スプシA列に「単語A,単語B,コメント」形式で1行1ペアが入っているスプシのGAS URLを指定
  • GAS側でカンマ分割して JSON で返すこと(下記の戻り値形式参照)
  • 手動データ貼り付け: GASを使わずに直接 CSV をテキストエリアに貼り付けても可

GAS の戻り値形式

[
  { "a": "ぶぎ", "b": "ぶじ", "comment": "...(不変あり,1文字違い)",
    "invariant": "yes", "anagram": "no", "diffCount": 1 },
  ...
]

invariant/anagram は "yes" | "no" | "unknown"、diffCount は数値または null(不明)。 ツール側でも生コメントから自動判定するので、GAS側で属性を返さなくても動作する。

検索条件

  • 文字 x と y を1文字ずつ指定
  • 位置 同(n=m): 単語Aの位置 i と単語Bの位置 i に x, y がある
  • 位置 異(n≠m): 単語Aの x の位置と単語Bの y の位置が異なる
  • 複数条件は AND/OR で結合
  • A↔B 入れ替えも自動で評価される(x→y と y→x 両方を見る)

フィルタ

  • 文字数: 単語の文字数(A=Bが前提)で範囲指定
  • 文字違い数: コメントから抽出した N 文字違いで範囲指定
  • 不変文字 / アナグラム: あり / なし / 不明 をチェックボックスで選択

補助機能

  • 同コメ ボタン: 結果行ごとに「同じコメント本文を持つ全ペア」をモーダルで表示。「このアンサーキーで他にどんな例が作れるか」を一覧
  • ズレ ボタン: 結果行のコメントから リスト1→リスト2 を読み取り、各位置のズレ数別にグループ化して表示。グループは Tier1 (引き算1値で結合する直接的なペア) / Tier2 (リスト全体を d 動かすことで結合できるペア) / 混在 に分類。ズレ0(完全一致)は紫背景で最優先強調。テーブル上のコメント着色は Tier1 由来のペアのみ反映
  • デバッグログ: 入力条件・処理結果を記録。問題報告時にコピーして貼り付け
  • 状態ハッシュ: Base64エンコードされた状態を読み込み・コピーで復元・共有
  • CSVコピー: 結果(フィルタ適用後)を CSV 形式でクリップボードへ
